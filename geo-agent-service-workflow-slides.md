# GEO Agent 서비스 안내
## 사이트 운영자를 위한 2가지 활용 워크플로우

### 요약본 (1페이지)

| 관점 | 목적 | 진행 워크플로우 | 주요 산출물 | 핵심 이득 |
|---|---|---|---|---|
| 1) GEO 평가 자료 수신 | 현재 사이트의 GEO 상태 진단 | 타깃 입력 → 자동 분석/검증 → 평가 리포트 수신 | GEO 점수, 경쟁 격차, 우선 개선 과제 | 빠른 현황 파악, 투자 우선순위 명확화 |
| 2) GEO 기반 사이트 개선 | 실제 콘텐츠/구조 개선안 도출 | 진단 → 전략 수립 → 로컬 클론 최적화 루프 → 검증/리포트 | Before/After 비교, 변경 이력, 적용 가능한 결과물(Archive) | 실행 가능한 개선안 확보, 반복 최적화로 성과 누적 |

> 공통 원칙: 원본 사이트는 직접 수정하지 않고, 로컬 클론에서 안전하게 개선안을 만든 뒤 운영자가 선택 반영

---

# GEO Agent가 제공하는 서비스 범위

## 무엇을 해주는가
- LLM 관점에서 사이트가 얼마나 잘 인식/인용되는지 다각도로 평가
- 경쟁 사이트 대비 부족한 정보 구조/콘텐츠 신호를 탐지
- 점수 개선을 위한 작업 우선순위와 변경안을 체계적으로 제시

## 어떻게 동작하는가 (공통 파이프라인)
`INIT → ANALYZING → CLONING → STRATEGIZING → OPTIMIZING → VALIDATING → REPORTING → COMPLETED`

## 사용자에게 중요한 설계 원칙
- **읽기 전용 원칙**: 원본 URL 직접 수정 없음
- **안전한 실험**: 로컬 클론 기반으로만 최적화 수행
- **검증 기반 의사결정**: 변경 전/후 점수와 근거를 함께 제공

---

# 관점 1: "평가 자료를 받아보는" 사용자 워크플로우

## 목표
- "우리 사이트가 GEO 관점에서 지금 어느 수준인지" 빠르게 파악
- 실행 전에 **현황 진단**과 **우선순위**부터 확보

## 단계별 진행
1. **대상 등록**: URL, 비즈니스/콘텐츠 맥락 입력
2. **자동 분석**: 구조/가독성/정보 인식/인용 가능성 평가
3. **경쟁 비교**: 주요 경쟁군 대비 강점/약점 분석
4. **리포트 수신**: 점수, 등급, 개선 후보 목록 확인

## 사용자 액션
- 내부 공유용 진단 자료로 활용
- "지금 당장 할 일"과 "보류 과제" 분리

---

# 관점 1: 상세 산출물 (받게 되는 자료)

## 1) Executive Summary
- 종합 GEO 점수 및 등급
- 핵심 리스크 3~5개
- 개선 기대효과(우선순위 포함)

## 2) 진단 리포트
- 차원별 점수(인용률/정확도/정보 인식/커버리지 등)
- 페이지/섹션별 취약 포인트
- 경쟁 사이트 대비 갭 분석

## 3) 실행 백로그
- 즉시 개선 항목 (Quick Wins)
- 구조 개선 항목 (중요도 높음)
- 검증이 필요한 실험 항목

## 4) 의사결정 지원 포맷
- 경영진용 요약
- 실무팀용 상세 체크리스트

---

# 관점 1: 기대 이득

## 비즈니스 측면
- 감으로 하던 SEO/GEO 논의를 **정량 근거 기반**으로 전환
- 리소스가 적어도 "효과 큰 순서"로 착수 가능

## 운영 측면
- 개발/콘텐츠/마케팅 팀 간 공통 기준 정렬
- 개선 과제의 우선순위 충돌 감소

## 리스크 측면
- 대규모 개편 전에 잠재 리스크 조기 탐지
- 잘못된 가설에 대한 과잉 투자 방지

---

# 관점 2: "GEO로 사이트를 개선하려는" 사용자 워크플로우

## 목표
- 단순 진단을 넘어, 실제 반영 가능한 개선안을 확보
- 반복 루프를 통해 점수와 성과를 지속적으로 끌어올림

## 단계별 진행
1. **진단 시작(ANALYZING)**: 현황 점수 + 개선 가설 도출
2. **클론 생성(CLONING)**: 원본 복제본에서 안전하게 작업 환경 구성
3. **전략 수립(STRATEGIZING)**: 변경 우선순위/작업 묶음 계획
4. **최적화 수행(OPTIMIZING)**: 콘텐츠/구조 신호 개선
5. **검증(VALIDATING)**: 점수 변화와 품질 검토
6. **반복 여부 결정**: 목표 미달 시 전략 단계로 루프백
7. **보고/전달(REPORTING)**: 운영 반영 가능한 결과 패키지 전달

---

# 관점 2: 최적화 루프의 실제 운영 방식

## 루프 제어 기준
- 목표 점수 도달 시 자동 중단
- 직전 대비 개선폭이 작으면(한계 효용) 중단
- 최대 반복 횟수 도달 시 중단
- 필요 시 사용자 수동 중단

## 각 사이클에서 확인하는 것
- 어떤 변경이 점수 향상에 기여했는지
- 어떤 변경은 영향이 미미했는지
- 다음 사이클에서 강화/폐기할 전략

## 핵심 장점
- "한 번에 완벽"이 아닌, **검증 가능한 점진 개선**
- 누적 학습으로 다음 실험의 정확도 향상

---

# 관점 2: 최종 전달물 (실행 가능한 형태)

## Before / After 리포트
- 점수 변화, 변경 근거, 예상 효과를 한눈에 비교

## 변경 이력(Change Record)
- 어떤 파일/영역을 어떻게 바꿨는지 추적 가능

## Archive 패키지
- `original/` 원본 기준 자료
- `optimized/` 최적화 결과물
- `diff/` 변경 차이
- `report.json` 기계 판독 가능한 결과 요약

## 운영 반영 방식
- 운영자가 내부 검토 후 선별 반영
- 배포 정책에 맞춰 단계적 적용 가능

---

# 관점 2: 기대 이득

## 성과 측면
- LLM 인용 가능성/정확도/커버리지 개선
- 경쟁군 대비 정보 전달력 향상

## 프로세스 측면
- 개선안의 타당성을 데이터로 검증 후 반영
- 변경 실패 비용을 클론 단계에서 흡수

## 조직 측면
- "무엇을 왜 바꿨는지" 기록이 남아 협업 효율 상승
- 재현 가능한 최적화 플레이북 축적

---

# 두 관점 비교: 언제 어떤 모드를 선택할까?

| 상황 | 추천 모드 |
|---|---|
| 현재 수준 파악이 우선, 빠른 의사결정이 필요 | 1) 평가 자료 수신 |
| 실행 가능한 개선안까지 필요, 내부 반영 의지/역량 보유 | 2) GEO 기반 개선 |
| 예산/인력이 제한적 | 1로 시작 후, 핵심 과제만 2로 확장 |
| 경쟁 심화/콘텐츠 규모 확대 국면 | 2를 기본 운영 모드로 전환 |

## 권장 도입 순서
`평가(1) → 파일럿 개선(2, 소규모) → 정기 개선 루프(2, 운영화)`

# 기대 효과 정리 (결론)

## 공통 가치
- 사이트의 GEO 적합성을 **가시화**하고 **측정 가능**하게 만듦

## 관점 1의 가치
- 빠르고 명확한 진단으로 우선순위 기반 의사결정 지원

## 관점 2의 가치
- 안전한 실험과 반복 검증을 통해 실질 개선안 축적

## 최종 메시지
- GEO Agent는 "점수표 제공 도구"를 넘어,  
  **진단 → 전략 → 실행 → 검증 → 보고**를 잇는 운영 체계다.
